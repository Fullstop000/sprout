# 自治工程代理工作流程

## 代理核心职责
TokenBurn自治代理以"自主设定目标-推进实现-记录结果"为核心循环，自动识别工作区需求并执行工程任务。

## 完整工作流程

### 1. 上下文收集
代理首先扫描工作区上下文，包括：
- Git状态与历史提交
- TODO/FIXME/BUG注释
- 现有文档与代码结构
- 之前的工程报告

### 2. 目标设定
基于上下文分析，代理自主选择当前最有价值的工程目标，优先级规则：
1. P0：影响功能正确性的BUG
2. P1：需要修复的已知缺陷（FIXME）
3. P2：功能迭代与代码优化（TODO）
4. P3：临时解决方案重构（HACK）
5. 文档完善与项目结构优化

### 3. 任务执行
代理生成可执行的action列表，支持以下操作类型：
- `search_web`: 搜索互联网获取技术方案
- `run_command`: 执行终端命令（需符合安全策略）
- `write_file`: 写入/覆盖文件内容
- `append_file`: 追加内容到文件

### 4. 结果记录
代理自动生成工程报告，记录：
- 目标完成情况
- 执行的action与结果
- 环境限制与后续建议

## 代理模块说明

### src/llm247/autonomous.py
自治代理的核心循环实现，负责目标设定、action执行与结果记录。

### src/llm247/context.py
工作区上下文收集模块，包括：
- Git状态扫描
- TODO/FIXME/BUG注释识别
- 文档与代码结构分析

### src/llm247/tasks.py
任务管理模块，负责：
- 任务优先级排序
- 默认任务模板构建
- 任务执行结果验证

## 示例任务场景
1. **测试完善**：自动识别测试代码缺失的断言逻辑并补充
2. **文档优化**：基于现有代码结构完善README与AGENTS文档
3. **任务跟踪**：扫描注释生成优先级任务列表并更新跟踪文档
